# Specification Rule

Every specification should satisfy the following.

Explain purpose.

Explain philosophy.

Explain architecture.

Explain internal flow.

Explain mathematical model where applicable.

Explain state transitions.

Explain recovery.

Explain performance.

Explain security.

Explain scalability.

Explain future compatibility.

Explain acceptance criteria.

Specifications should describe engineering decisions.

Implementation details belong inside source code.
