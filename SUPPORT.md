# Support

## Getting Help

If you need help with MyDash, there are several resources available:

### Documentation

- **[ARCHITECTURE.md](ARCHITECTURE.md)** — System architecture and design decisions
- **[docs/](./docs/)** — Detailed implementation documentation
- **[.env.example](.env.example)** — Environment variable reference

### Issues

- **[Bug Reports](https://github.com/akaanakbaik/mydash-vps/issues/new?template=bug_report.md)** — Report bugs and unexpected behavior
- **[Feature Requests](https://github.com/akaanakbaik/mydash-vps/issues/new?template=feature_request.md)** — Suggest new features or improvements

### Community

- **GitHub Discussions** — Available in the repository
- **GitHub Issues** — For bug reports and feature requests

### Before Opening an Issue

1. Check the existing issues to see if your problem has already been reported
2. Make sure you're using the latest version
3. Include as much relevant information as possible (logs, screenshots, steps to reproduce)

### Response Times

We aim to respond to issues within:

- **Bug reports**: 2-3 business days
- **Feature requests**: 3-5 business days
- **Security reports**: Within 48 hours (see [SECURITY.md](SECURITY.md))
